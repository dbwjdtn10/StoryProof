# StoryProof-main vs StoryProof-ui 비교 분석

## 1. 개요
사용자께서 지적하신 `StoryProof-main`과 `StoryProof-ui`의 소설 내용 정렬 및 씬 구분 차이에 대한 기술적 분석 결과입니다.

## 2. 주요 차이점 점검

### A. 에디터 기술 스택 (Editor Stack)
*   **StoryProof-main**: 브라우저 기본 `textarea`를 기반으로 합니다. 텍스트 데이터 그 자체를 보여주기 때문에 레이아웃이 매우 안정적이고 예측 가능합니다.
*   **StoryProof-ui**: `Tiptap` (Rich Text Editor) 프레임워크를 사용합니다. 다양한 편집 기능(강조, 색상, 이미지 등)을 제공하지만, 내부적으로 HTML 요소를 관리하므로 CSS 스타일링이 어긋날 경우 정렬이 흐트러져 보일 수 있습니다.

### B. 씬(Scene) 시각화 방식
*   **StoryProof-main**:
    - 각 씬을 독립적인 카드 형태로 시각화.
    - `Scene X` 라벨이 별도의 고정 위치에 디자인되어 있어 씬 전환이 눈에 확 띕니다.
    - 씬 간 간격이 `60px`로 고정되어 있어 여백의 미가 살아있습니다.
*   **StoryProof-ui**:
    - 씬 구분 기능은 있으나, Tiptap 에디터의 영역과 겹쳐 보이거나 라벨의 시각적 대비가 낮아 씬이 하나로 뭉쳐 보일 수 있습니다.

### C. 데이터 전처리 (Normalization)
*   `StoryProof-main`은 데이터를 불러올 때 연속된 줄바꿈을 자동으로 정리하는 로직이 더 강력하게 적용되어 있어, 원본 텍스트의 복잡함에 상관없이 일정한 간격을 유지합니다.

## 3. 요약 비교표

| 항목 | StoryProof-main | StoryProof-ui |
| :--- | :--- | :--- |
| **정렬 상태** | 매우 정갈함 (고정 방식) | 가변적임 (에디터 설정 영향) |
| **씬 구분** | 명확함 (독립 블록화) | 모호함 (에디터 내 합쳐짐) |
| **글꼴/간격 설정** | 고정됨 | 사용자 조절 가능 (복잡도 증가) |
| **주요 용도** | 내용 확인 및 기본 교정 | 정밀 편집 및 독서 경험 커스텀 |

## 4. 해결 방향
`StoryProof-ui`의 강력한 기능을 유지하면서 `main`과 같은 정갈한 느낌을 주려면, `NovelEditor`의 하위 CSS 스타일(Padding, Margin, Line-height)을 `StoryProof-main`의 값으로 역이식하는 과정이 필요합니다.
