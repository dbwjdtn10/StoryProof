# StoryProof 툴바 및 에디터 기능 이관 가이드

이 문서는 작업한 **'작가 모드/독자 모드 툴바 분리'** 및 **'에디터 통합'** 기능을 다른 초기 프로젝트 작업본에 그대로 옮기기 위한 가이드입니다.

---

## 1. 필수 라이브러리 설치 (package.json)

상대방의 프로젝트 터미널에서 아래 명령어를 실행하여 필수 패키지를 설치해야 합니다.

```bash
npm install @tiptap/react @tiptap/starter-kit @tiptap/extension-underline @tiptap/extension-color @tiptap/extension-text-style @tiptap/extension-highlight @tiptap/extension-font-family @tiptap/extension-text-align lucide-react @mui/material @emotion/react @emotion/styled
```

---

## 2. 복사해야 할 파일 목록

아래 파일들을 현재 프로젝트의 경로와 동일하게 상대방 프로젝트로 복사하세요.

### ① 컴포넌트 (frontend/src/components/)
- `AuthorToolbar.tsx` (작가용 툴바 - **신규**)
- `ReaderToolbar.tsx` (독자용 툴바)
- `ChapterDetail.tsx` (메인 페이지 - **로직 포함**)
- `NovelEditor.tsx` (에디터 본체)
- `SettingsModal.tsx` (환경설정 창)
- `ui/` 폴더 전체 (버튼, 팝오버 등 기본 UI 요소)

### ② 스타일 (frontend/src/)
- `novel-toolbar.css` (툴바 디자인 핵심)
- `index.css` (전체 레이아웃 스타일)

---

## 3. 아이콘 폰트 설정 (index.html)

작가 툴바의 아이콘(Material Symbols)을 표시하기 위해 반드시 필요합니다.
**`frontend/public/index.html`** 파일의 `<head>` 태그 안에 아래 코드를 추가하세요.

```html
<!-- Material Symbols for Author Toolbar Icons -->
<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined:opsz,wght,FILL,GRAD@20..48,100..700,0..1,-50..200" />
```

---

## 4. 이관 시 주의사항

1.  **임포트 경로 확인**: `ChapterDetail.tsx`나 `AuthorToolbar.tsx` 상단에서 다른 파일(예: `api/novel.ts`)을 불러오는 경로가 상대방 프로젝트와 다를 경우, 해당 경로를 수정해야 합니다.
2.  **환경변수 확인**: `SettingsModal.tsx`에서 사용하는 API 호출 주소나 토큰 관리 로직이 상대방의 백엔드 설정과 일치하는지 확인하세요.
3.  **불필요한 파일 삭제**: 기존에 있던 `NovelToolbar.tsx`나 `WritingToolbar.tsx`는 더 이상 사용되지 않으므로 삭제하여 혼선을 방지하는 것이 좋습니다.

---

## 5. 적용 확인 방법

1.  파일 이관 및 라이브러리 설치 완료 후 `npm start` 실행.
2.  에디터 페이지(ChapterDetail)에 진입.
3.  상단 툴바가 현재 모드(작가/독자)에 맞게 나타나는지 확인.
4.  툴바 우측의 **톱니바퀴(환경설정)** 버튼을 눌러 모달창이 정상적으로 뜨는지 확인.
